# Reflexión Agile

## (a) Diferencia entre Scrum y Kanban

Scrum es una metodología que organiza el trabajo en ciclos llamados sprints, con roles definidos y reuniones periódicas para planificar y revisar el progreso. En cambio, Kanban es un sistema más flexible que utiliza un tablero visual para gestionar tareas en diferentes estados sin necesidad de trabajar por iteraciones fijas. Para un proyecto individual, me parece más útil Kanban porque es más simple de implementar, no requiere roles ni reuniones, y permite avanzar a mi propio ritmo organizando las tareas de manera visual.

## (b) ¿Qué es un MVP?

Un MVP (Producto Mínimo Viable) es una versión básica de un producto que incluye solo las funcionalidades esenciales para que funcione y pueda ser utilizado. La idea es lanzar algo simple pero útil, y luego mejorarlo con el tiempo. Si tuviera que hacer un sitio web para un negocio local, primero crearía una página sencilla con información básica como el nombre del negocio, servicios, ubicación y un formulario de contacto. Después, agregaría mejoras como diseño más avanzado, galería de imágenes o integración con redes sociales.

## (c) Uso del tablero Kanban

El tablero Kanban me ayudó a organizar mejor las tareas del trabajo práctico ya que directamente colocaba las cosas generales que tenia que hacer, por ejemplo: crear un archivo formulario.html o darle estilo a dicho estilo. Aun no manejo bien el tablero pero es bueno ir agregando buenas costumbres de trabajo.
